# QA Checklist

## 必过项

- 是 16:9 横版。
- 背景是干净白底。
- 有白煮蛋 IP。
- 白煮蛋承担核心动作，不只是装饰。
- 有简单深色围裙，且围裙清楚可见。
- 围裙在主体正面，面积足够像围裙，不是腰线；至少有小结、系带、细竖纹或小口袋之一。
- 有细手细脚和小工作鞋。
- 没有眼镜。
- 默认形态是白煮蛋；使用变体时仍能看出“围裙蛋小工友”。
- 蛋壳裂纹、碎壳、蛋黄、蛋白、蛋花或蒸汽服务动作和情绪，不是随机装饰。
- 没有复刻旧案例构图，而是为当前文章生成了新隐喻。
- 画面怪诞、有创意、有意思。
- 简洁清爽，主体不超过画面约 60%。
- 一张图只讲一个核心结构。
- 中文标注少、短、能读。
- 橙色只用于主路径或箭头。
- 红色只用于重点、问题、提醒或结果。
- 蓝色只用于补充说明、反馈或系统状态。
- 可靠交付时，图片必须真实存在于 workspace 文件路径中。
- 单张图片目标大小不超过 100KB。
- 单张图片硬限制小于 1MB。
- 视觉质量优先任务不能用本地脚本草图冒充 AI 插图。

## 失败信号

出现以下情况，重生成或局部编辑：

- 左上角有“常见坑 / Workflow / 系统架构图 / 路线图”等标题。
- 白煮蛋像吉祥物、表情包或儿童卡通。
- 戴了眼镜。
- 没有围裙，或围裙被蛋黄/蛋白/道具遮住。
- 围裙太小、太淡、像腰线、像普通黑块，或没有任何小结/系带/纹路/口袋细节。
- 小工作鞋消失，只剩两根细腿。
- 变体变成普通荷包蛋、炒蛋、蒸蛋或蛋花，看不出主体。
- 蛋黄、蛋液、碎壳画得脏乱或恶心。
- 画面像 PPT、课程课件、正式流程图。
- 元素太多、箭头太多、节点太多。
- 文字变成大段解释。
- 背景有纸纹、阴影、渐变、米色、噪点。
- 真实 UI 截图或科技感界面。
- 中文错字严重或标注不可读。
- 画面太死板，没有荒诞隐喻。
- 只有会话预览，没有本地文件。
- 单张文件超过 1MB。
- 用户要正式插图，但输出像本地模型/脚本生成的线段草图。

## 迭代方法

- 太普通：让白煮蛋成为动作主体，加入一个奇怪但成立的隐喻。
- IP 不够稳：先把正面深色围裙、小结/系带/细竖纹/小口袋、黑点眼、细手细脚、小工作鞋补回来。
- 变体太散：保留最大主体蛋块戴围裙，其他蛋块只做情绪碎片。
- 太复杂：删节点，只保留一个动作和 3-5 个短标注。
- 太可爱：减少腮红、爱心、大笑脸，强调 restrained neutral expression、not mascot。
- 太 PPT：去掉标题、边框、整齐网格和过多箭头，改成手绘场景。
- 文字错：优先局部编辑；错得多就重生成并减少标注数量。
- 文件过大：先用 PNG optimize 和高压缩等级；仍超过 100KB 时减少细碎元素或量化到 32-64 色；超过 1MB 必须重做或简化。
- 画面像本地草图：切回视觉质量优先模式，用 AI 生图或可落盘 API/CLI；只有用户明确要求确定性小文件时才继续用本地脚本。

## 交付判断

高质量图应该让读者先觉得“有点怪”，然后 1 秒内看懂结构。

如果第一眼像教程页，而不是白纸上的怪诞产品草图，就不合格。
