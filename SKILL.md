# 白煮蛋怪诞正文配图

## 核心定位

为中文文章设计和生成 16:9 横版正文配图。目标不是做商业插画、PPT 信息图或可爱卡通，而是把文章里的关键判断、流程、结构、状态或隐喻，变成一张清爽、怪诞、有创意、可读但不说明书的手绘解释图。

默认视觉 IP 是“白煮蛋”：一个空心白色鸡蛋形小人，黑色手绘轮廓，黑点眼，克制表情，细手细脚。稳定视觉钩子是简单深色围裙和小工作鞋。A 形态（完整围裙白煮蛋）是主视觉和默认选择。B/C/D/E 等变体只在文章内容和蛋形态语义强相关时使用，不能为了变化而特地切换。无论变成荷包蛋、炒蛋、蒸鸡蛋、蛋花、溏心蛋，围裙都必须保留，让它始终是“围裙蛋小工友”。

## 先读这些参考

按任务需要读取，不要一次塞满上下文：

- `references/style-dna.md`：风格 DNA、颜色、文字、禁忌。
- `references/egg-ip.md`：白煮蛋 IP 的默认形象、变体规则、情绪系统和禁忌。
- `references/composition-patterns.md`：结构类型、原创隐喻方法和反复刻规则。
- `references/prompt-template.md`：单张生图提示词模板。
- `references/qa-checklist.md`：生成后检查和迭代规则。

## 工作流

### 1. 消化正文

先读用户给的正文、链接、Notion 页面、Markdown 文件或截图内容。提炼：

- 核心观点是什么
- 哪些段落承担认知转折
- 哪些内容适合用图解释
- 哪些地方只适合文字，不需要图

不要平均配图。优先选择“认知锚点”，例如：核心判断、两个断点、输入输出闭环、分流、前后对比、承接路径、常见坑、角色状态变化、结构重组。

### 2. 先出配图策略

如果用户只是说“分析怎么配图 / 思考哪些地方需要配图”，先给 shot list。每张图写清楚：

- 放在哪个段落后
- 图的主题
- 核心意思
- 结构类型
- 白煮蛋在图里做什么
- 是否需要蛋形态变体
- 建议元素
- 建议中文标注词

默认 4-8 张。文章很短时 1-3 张；长文也不要轻易超过 9 张。

### 3. 单张生成

如果用户明确要求“生成 / 输出 / 做图 / 帮我生成”，不要停下来等确认。需要可靠交付文件时，默认用本地脚本、SVG、PIL 或 canvas 生成确定性 PNG，不依赖内置 `image_gen` 自动落盘。内置 `image_gen` 只用于方向预览或用户明确要求 AI 生图预览的场景；预览成功不等于本地文件已保存。

每张图单独生成，不要把多张图拼在一张里。

每张图只讲一个核心结构。提示词必须包含：

- 16:9 横版中文正文配图
- 纯白背景
- 黑色手绘线稿
- 少量红色/橙色/蓝色中文手写批注
- 大量留白
- 白煮蛋作为核心动作主体
- 简单深色围裙和小工作鞋作为稳定识别点
- 蛋壳裂纹、碎壳、蛋黄、蛋白、蛋花等只服务动作和情绪
- 禁止 PPT、商业插画、幼稚可爱、复杂架构、左上角类型标题

默认形态是 A：完整的围裙白煮蛋小工友。只有当内容隐喻强烈需要“摊开、打散、凝固、外溢、重组、流动”时，才使用 B/C/D/E 等荷包蛋、炒蛋、蒸鸡蛋、蛋花、溏心蛋变体。除非强相关，否则不要为了视觉丰富度切换变体；一张图最多选择一个主变体，其他蛋状态只能作为少量情绪细节。变体必须保留围裙。

### 4. 检查与迭代

生成后检查 `references/qa-checklist.md`。如果出现以下问题，优先重生成或局部编辑：

- 白煮蛋只是装饰
- 没有围裙或鞋子
- 蛋变体失去主体识别
- 画面太满
- 太像流程图/PPT
- 中文太多或错字严重
- 左上角出现“常见坑/流程图/系统架构图”等标题
- 画风太可爱、幼稚、死板
- 背景不是干净白底

### 5. 保存交付

如果用户在 workspace 内工作，把最终图复制到：

```text
assets/<article-slug>-illustrations/
```

按顺序命名：

```text
01-topic-name.png
02-topic-name.png
```

保留原始生成文件，不要覆盖已有资产，除非用户明确要求替换。

文件质量限制：

- 默认输出 16:9 PNG。
- 单张目标大小控制在 100KB 以内。
- 单张硬上限为 1MB；超过 1MB 不允许交付，必须压缩、简化画面或降低颜色数量后重新输出。
- 生成后必须检查文件实际大小；不要只依赖预览判断。
- 对纯白手绘线稿，优先使用 PNG `optimize=True`、高压缩等级和有限颜色量化来控体积。

## 输出口径

生成前的策略输出要短而准。生成后的交付要包含：

- 生成了几张
- 每张图的用途
- 保存路径
- 每张图的文件大小是否满足 100KB 目标和 1MB 硬限制
- 哪些图最稳，哪些图是可选

不要长篇解释风格理论；让图自己说话。
