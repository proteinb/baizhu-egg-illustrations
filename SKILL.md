# 白煮蛋怪诞正文配图

## 核心定位

为中文文章设计和生成 16:9 横版正文配图。目标不是做商业插画、PPT 信息图或可爱卡通，而是把文章里的关键判断、流程、结构、状态或隐喻，变成一张清爽、怪诞、有创意、可读但不说明书的手绘解释图。

默认视觉 IP 是“白煮蛋”：一个空心白色鸡蛋形小人，黑色手绘轮廓，黑点眼，克制表情，细手细脚。稳定视觉钩子是简单深色围裙和小工作鞋。A 形态（完整围裙白煮蛋）是主视觉和默认选择。B/C/D/E 等变体只在文章内容和蛋形态语义强相关时使用，不能为了变化而特地切换。每组 4-8 张长文配图中，默认以 A 为主；如果文章存在强相关语义，可选择 1 张使用 B/C/D/E 作为主变体，且必须在 Shot List 中写明选择理由。无论变成荷包蛋、炒蛋、蒸鸡蛋、蛋花、溏心蛋，围裙都必须保留，让它始终是“围裙蛋小工友”。

白煮蛋 IP 有一组不可省略的视觉锁定项：正面深色围裙、前方小结或系带、细竖纹或小口袋、小工作鞋、黑点眼、细手细脚、克制空表情。围裙不能只是腰上一条线，不能被蛋黄、蛋白、道具、文字或构图遮住；如果看不清围裙和鞋子，这张图不合格。

如果用户要求“视频化、分层、透明 PNG、动效、剪辑、manifest、口播稿对齐、unit_id”等输出，本 skill 仍保持同一套白煮蛋画风和 Shot List 标准，只额外输出可追溯的语义模块、图层建议和 manifest。视频化分层不是把画面做成 PPT 零件表，而是让同一张白煮蛋配图可以被后期拆层、入场、遮挡、轻微运动和复用。

## 工作流

### 1. 消化正文

先读用户给的正文、链接、Notion 页面、Markdown 文件或截图内容。提炼：

- 核心观点是什么
- 哪些段落承担认知转折
- 哪些内容适合用图解释
- 哪些地方只适合文字，不需要图

如果上游提供了 `unit_id`，先识别并保留原始编号：

- `unit_id` 是口播稿语义单元的唯一主键，只能消费、透传和引用。
- 不允许重新生成另一套语义编号。
- 不允许用 `图 N`、段落序号、图片序号替代 `unit_id`。
- 每个视觉模块和视觉元素都必须能追溯到一个或多个 `unit_id`。
- 如果没有上游 `unit_id`，继续按原文章配图流程生成 Shot List，但标记 `unit_id_source: missing`；不能伪造已对齐口播稿的 `unit_id`。

不要平均配图。中文长文配图必须先做 Shot List，不逐段画。优先选择“认知锚点”，例如：核心判断、两个断点、输入输出闭环、分流、前后对比、承接路径、常见坑、角色状态变化、结构重组。

工具清单类文章直接使用“三图法”：

- 入口选择图：读者面对多个工具/路径时，先帮他选入口。
- 流程接力图：展示工具之间如何传递任务、素材、判断或结果。
- 结果复用图：展示最后产物如何被二次使用、分发、沉淀或变成资产。

### 2. 先出配图策略

如果用户只是说“分析怎么配图 / 思考哪些地方需要配图”，先给 shot list。每张图写清楚：

- 文章主题
- 认知转折点
- 关键动作
- 画面构图
- 语义模块
- 批注词
- 分层建议
- 禁忌项

默认 4-8 张。文章很短时 1-3 张；长文也不要轻易超过 9 张。

Shot List 固定模板：

```text
图 N：<短主题>
覆盖 unit_id：<u001 / u002 / u003，若无上游 unit_id 则写 missing>
文章主题：<这张图覆盖的文章主题>
认知转折点：<读者在这里需要理解的变化、冲突或新判断>
关键动作：<白煮蛋正在做的物理动作>
画面构图：<主体、道具、箭头、空间关系>
语义模块：<列出 module_id、source_unit_ids、module_role、模块内元素>
IP 锁定：<正面深色围裙清楚可见，有小结/系带/细竖纹或小口袋；小工作鞋可见；黑点眼、细手细脚、克制空表情保留>
批注词：<3-6 个短中文词>
分层建议：<哪些对象应作为独立透明 PNG 图层，哪些对象必须作为同一模块整体保留>
禁忌项：<这张图不能画什么>
```

视频化分层任务必须按 `references/video-layered-output.md` 输出 `module_id`、元素层级和 manifest；普通正文配图任务可以只给 Shot List 和单图。

工具清单类文章默认只给 3 张，不按工具数量逐个画：

```text
图 1：入口选择图
图 2：流程接力图
图 3：结果复用图
```

### 3. 单张生成

如果用户明确要求“生成 / 输出 / 做图 / 帮我生成”，不要停下来等确认。先判断生成模式，不要把所有图片都交给本地脚本。

生成模式：

- **视觉质量优先**：用户要“配图、插图、主视觉、好看、正式发布图、风格效果”时，优先使用 AI 生图能力生成真正的插图效果。本地脚本不能替代这一模式，因为它更像草图/示意图。
- **可靠小文件优先**：用户明确要求“可靠落盘、确定性、本地生成、100KB 内、批量稳定交付、不要依赖预览”时，才使用本地脚本、SVG、PIL 或 canvas 生成确定性 PNG。
- **混合模式**：先用 AI 生图确定视觉方向，再用本地脚本复刻成可控小文件。只有当用户同时要求视觉质量和严格文件大小时使用。

内置 `image_gen` 的预览成功不等于本地文件已保存。如果需要 AI 级视觉质量且必须可靠落盘，应优先使用可指定输出路径的 CLI/API 流程；如果没有可用 API/CLI，就明确说明只能提供预览或改用本地确定性草图。

每张图单独生成，不要把多张图拼在一张里。

每张图只讲一个核心结构。提示词必须包含：

- 16:9 横版中文正文配图
- 纯白背景
- 黑色手绘线稿
- 少量红色/橙色/蓝色中文手写批注
- 大量留白
- 白煮蛋作为核心动作主体
- 简单深色围裙和小工作鞋作为稳定识别点；围裙必须在主体正面清楚可见，有小结/系带/细竖纹或小口袋，不能缩成一条腰线
- 蛋壳裂纹、碎壳、蛋黄、蛋白、蛋花等只服务动作和情绪
- 禁止 PPT、商业插画、幼稚可爱、复杂架构、左上角类型标题

默认形态是 A：完整的围裙白煮蛋小工友。只有当内容隐喻强烈需要“摊开、打散、凝固、外溢、重组、流动”时，才使用 B/C/D/E 等荷包蛋、炒蛋、蒸鸡蛋、蛋花、溏心蛋变体。除非强相关，否则不要为了视觉丰富度切换变体；一张图最多选择一个主变体，其他蛋状态只能作为少量情绪细节。每组 4-8 张长文配图中最多 1 张使用 B/C/D/E 主变体；如果使用，必须在 Shot List 中写明“变体选择理由”。没有明确理由则保持 A。变体必须保留围裙。

生成提示词必须显式写出 IP 锁定项。不要只写“白煮蛋小人”，要写“正面深色围裙清楚可见，有小结/系带/细竖纹或小口袋，小工作鞋可见，黑点眼，细手细脚，克制空表情”。如果使用本地模型，更要重复一次“apron front-center, visible, not hidden, not a belt line”。

### 4. 检查与迭代

生成后检查 `references/qa-checklist.md`。如果出现以下问题，优先重生成或局部编辑：

- 白煮蛋只是装饰
- 没有围裙或鞋子
- 围裙太小、像腰线、没有小结/系带/细竖纹/小口袋，或被道具遮住
- 蛋变体失去主体识别
- 画面太满
- 太像流程图/PPT
- 中文太多或错字严重
- 左上角出现“常见坑/流程图/系统架构图”等标题
- 画风太可爱、幼稚、死板
- 背景不是干净白底

### 5. 保存交付

如果用户在 workspace 内工作，把最终图复制到：

```text
assets/<article-slug>-illustrations/
```

按顺序命名：

```text
01-topic-name.png
02-topic-name.png
```

保留原始生成文件，不要覆盖已有资产，除非用户明确要求替换。

如果是视频化分层交付，还要在同一目录保存：

```text
manifest.json
layers/
```

`manifest.json` 必须记录 `unit_id_source`、每张图覆盖的 `unit_id`、每个 `module_id`、每个图层文件、图层用途、建议入场方式和依赖关系。图层文件名不得只用 `图1` 或段落序号，要包含 `unit_id` 或 `module_id`，例如 `u003_m02_apron-egg-worker.png`。没有上游 `unit_id` 时，manifest 写 `unit_id_source: "missing"`，并用 `source_unit_ids: []`，不要编造 `u001`。

文件质量限制：

- 默认输出 16:9 PNG。
- 可靠小文件模式下，单张目标大小控制在 100KB 以内。
- 可靠小文件模式下，单张硬上限为 1MB；超过 1MB 不允许交付，必须压缩、简化画面或降低颜色数量后重新输出。
- 视觉质量优先模式下，不要为了 100KB 目标牺牲画面质量；先交付高质量图，再按用户要求另出压缩版。
- 生成后必须检查文件实际大小；不要只依赖预览判断。
- 对纯白手绘线稿，优先使用 PNG `optimize=True`、高压缩等级和有限颜色量化来控体积。

## 输出口径

生成前的策略输出要短而准。生成后的交付要包含：

- 生成了几张
- 每张图的用途
- 保存路径
- 每张图的文件大小是否满足 100KB 目标和 1MB 硬限制
- 哪些图最稳，哪些图是可选
- 如果是视频化分层任务：`manifest.json` 路径、图层数量、哪些图层可独立运动、哪些模块必须整体保留

不要长篇解释风格理论；让图自己说话。
